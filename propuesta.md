1.  **Módulo de Gestión de Sucursales**

## Propósito Permitir al usuario Maestro administrar las sucursales de la red de gimnasios, gestionando su creación, modificación, eliminación y consulta. {#propósito-permitir-al-usuario-maestro-administrar-las-sucursales-de-la-red-de-gimnasios-gestionando-su-creación-modificación-eliminación-y-consulta.}

## Funciones Principales

- **Iniciar Sesión**: Permite al usuario Maestro acceder al sistema con credenciales específicas (\"M4st3rAxF\" y \"Us3rM4st3rAxF\") para gestionar sucursales, es decir a los usuarios Sucursal.

- **Agregar Sucursales**: Facilita la creación de nuevas sucursales mediante el llenado de los campos Nombre Sucursal, Dirección, Código Postal, Usuario y Contraseña.

- **Eliminar Sucursales**: Permite la eliminación de sucursales existentes del sistema.

- **Modificar Sucursales**: Habilita la edición de la información de sucursales registradas, actualizando los campos Nombre Sucursal, Dirección, Código Postal, Usuario y Contraseña.

- **Buscar Sucursales**: Ofrece la posibilidad de consultar todas las sucursales registradas y visualizar sus detalles completos.

- **Cerrar Sesión**: Finaliza la sesión del usuario Maestro, retornando a la pantalla inicial del sistema.

2\. **Módulo de Gestión de Sucursal**

**Propósito  
**Permitir al usuario Sucursal administrar el personal, suscripciones, promociones, y reportes de incidencias de una sucursal específica, así como comunicarse con el personal mediante avisos, utilizando credenciales asignadas por el usuario Maestro.

**Funciones Principales**

- **Iniciar Sesión**: Permite al usuario Sucursal acceder al sistema con las credenciales (Usuario y Contraseña) asignadas por el usuario Maestro, identificando su rango para otorgar los permisos correspondientes.

- **Mostrar el historial de acceso de los suscriptores**: Se puede visualizar por días el acceso de los suscriptores, hora, fecha y método de acceso

- **Agregar Personal**: Agregar Personal: Facilita el registro de nuevos empleados rellenando los campos Nombres, Apellido Paterno, Apellido Materno, Edad, Usuario, Contraseña, Puesto (seleccionado de una lista desplegable: staff, entrenador, nutriólogo, entrenador/nutriólogo), Sexo, Sucursal y Fotografía. El campo Fotografía permite cargar una imagen del empleado para su identificación en reportes de incidencias generados por los suscriptores.

- **Eliminar Personal**: Permite la eliminación de usuarios Personal registrados en la sucursal.

- **Modificar Personal**:Habilita la edición de la información de empleados existentes, actualizando cualquier campo registrado, incluyendo la Fotografía del empleado, que puede ser reemplazada.

- **Buscar Personal**: Ofrece la consulta de usuarios Personal dados de alta, mostrando todos sus campos registrados.

- **Gestionar Suscripciones**: Permite agregar nuevos tipos de suscripciones rellenando los campos Duración, Nombre, Precio, límite de sesiones de nutriologo, límite de sesiones de entrenador, así como eliminar, modificar o buscar suscripciones existentes y visualizar sus detalles (Esto refiriéndose al tiempo, como mensual, pase diario, anual, etc, los unicos diferentes "beneficios" será el límite de sesiones de entrenador y de nutriologo).

- **Gestionar Promociones**: Permite la creación de promociones rellenando los campos Duración, Nombre, Descripción, límite de sesiones de nutriologo, límite de sesiones de entrenador, Precio, con opciones para eliminar, modificar o buscar promociones registradas y consultar sus detalles, además, pueden crearse promociones únicamente que cuenten con sesiones de atención del nutriólogo y/o entrenador, sin necesidad de agregar días de suscripción adicionales, estas sesiones de atención siempre estarán vigentes y no caducan hasta ser usados, pero solo se pueden usar mientras se tenga una suscripción activa.

- **Enviar Avisos**: Permite enviar mensajes al personal seleccionando uno o varios destinatarios para comunicar información relevante.

- **Acceso al módulo Análisis de Incidencias:**Permite al usuario Sucursal acceder a una sección dedicada en la interfaz web para consultar los informes generados automáticamente por el Módulo de Análisis de Incidencias, además permite configurar la frecuencia con la que desea recibir estos reportes, puede ser en un rango de días, semanas o inclusive meses, es a elección del usuario sucursal.

- **Cerrar Sesión**: Finaliza la sesión del usuario Sucursal, retornando a la pantalla inicial del sistema.

- **Configuración de recompensas**: El sistema le permite agregar una nueva recompensa, ingresando 'nombre' y 'costo' de las recompensas, estas recompensas aparecerán a los usuarios para que ellos puedan ayudar a los suscriptores a canjearlas.

## Funciones Principales

- **Acceder al Módulo de Usuarios**: Redirige al módulo de Usuarios al hacer clic en el bloque correspondiente en la pantalla principal.

- **Acceder al Módulo de Gestión de Suscripciones**: Redirige al módulo de Gestión de Suscripciones al seleccionar el bloque respectivo.

- **Mostrar Estadísticas de Suscriptores**: Presenta en la pantalla principal el total de usuarios con suscripciones activas e inactivas en la sucursal.

- **Mostrar el historial de acceso de los suscriptores**: Se puede visualizar por días el acceso de los suscriptores, hora, fecha y método de acceso.

- **Acceder al Módulo de Entrenamiento**: Habilita el acceso al módulo de Entrenamiento si el usuario tiene rango de entrenador; si no, requiere credenciales de un entrenador autorizado.

- **Cerrar Sesión**: Termina la sesión del usuario Personal, devolviendo el sistema a la pantalla inicial.

##  5. Módulo de Gestión de Alertas y Escalada de Reportes(Módulo de Reportes) {#módulo-de-gestión-de-alertas-y-escalada-de-reportesmódulo-de-reportes}

Propósito:

Asegurar que todos los reportes generados por los suscriptores reciban la atención y el seguimiento adecuados por parte del personal del gimnasio. Implementar un sistema de alertas escalonado para notificar a los usuarios relevantes, esto mediante una campanita que agregara un número marcado en rojo junto a ella indicando el número de notificaciones, en caso de no haber solo se ve la campanita sin ningún indicador de números, además escalar la prioridad de un reporte si no se le da seguimiento oportuno, garantizando que ninguna solicitud quede desatendida.

**Funciones Principales:**

- **Monitoreo de Reportes:** El sistema monitorea el estado de los reportes abiertos para identificar aquellos que no han recibido seguimiento en un tiempo predefinido o que solo han sido marcados como \"recibidos\" sin progreso.

- **Sistema de Alertas Escalable (Strikes):**

  - **Primer Strike (Alerta a Nivel de Sucursal - Personal):** Si un reporte no avanza en las primeras 24 horas que se recibe el reporte, el sistema enviará una notificación interna de \"primer strike\" a *todos los usuarios de tipo \'Personal\'* asignados a la sucursal del reporte. Esta notificación servirá como un recordatorio para tomar acción.

  - **Segundo Strike (Alerta a Nivel de Sucursal - Usuario Sucursal):** Si el reporte continúa sin seguimiento después del primer strike por otras 24 horas (siendo 48 en total desde el inicio del reporte) , el sistema emitirá un \"segundo strike\". Esta alerta incluirá una nueva notificación a *todos los usuarios de tipo \'Personal\'* de la sucursal, y adicionalmente, enviará una notificación al *usuario \'Sucursal\'* (equivalente a un gerente o administrador de la sucursal).

  - **Tercer Strike (Escalada Máxima - Prioridad Alta):** En caso de que el reporte persista sin resolución tras el segundo strike por otras 24 horas (siendo 72 horas en total desde el inicio del reporte) , se activará el \"tercer strike\". Esto desencadenará las siguientes acciones:

    - Envío de un **correo electrónico y una notificación de alta prioridad** al *usuario \'Sucursal\'*.

    - Notificación al *suscriptor* a través de la aplicación móvil, informando sobre el número de strikes que ha alcanzado su reporte y explicando brevemente el significado de cada strike (e.g., \"Su reporte ha activado una alerta de escalada interna para asegurar su pronta resolución\").

    - **Habilitación de la Opción de Reenvío Directo al Usuario Sucursal:** Se habilitará una funcionalidad en la aplicación móvil del suscriptor que le permitirá reenviar el reporte directamente al *usuario \'Sucursal\'* (gerente), asegurando que sea atendido de forma prioritaria.

- **Historial de Alertas:** El sistema deberá mantener un registro de todos los \"strikes\" generados para cada reporte, incluyendo la fecha, hora y los usuarios notificados.

- **Flexibilidad de Actualización del Reporte:** Durante todo el proceso de escalada y alertas, el *usuario \'Personal\'* que inicialmente gestiona el reporte no perderá la capacidad de actualizar el estado del caso y continuar con su resolución. Esto permite que el personal en turno, o cualquier otro usuario \'Personal\' de la sucursal, pueda tomar la responsabilidad del reporte y darle seguimiento hasta su resolución, incluso si el reporte ha sido escalado.

- **Configuración de Tiempos:** El sistema deberá permitir la configuración de los tiempos de inactividad que disparan cada nivel de \"strike\" para adaptarse a las necesidades específicas de cada tipo de reporte o política del gimnasio.

- **Interfaz de Reportes Escaldados:** Una sección dedicada dentro del módulo de reportes donde el *usuario \'Sucursal\'* pueda visualizar de forma prioritaria todos los reportes que han alcanzado el \"tercer strike\" o que han sido reenviados directamente, facilitando su gestión prioritaria.

  - Fila 2: Suscripción Anual, Inicio: 16/07/2025, Fin: 15/07/2026

- El sistema SOLAMENTE dará acceso cuando se intente acceder en una fecha DENTRO del rango de la suscripción, en este caso ficticio, desde 16/06/2025 hasta 15/07/2026.

- **Calcular Macronutrientes**: Genera automáticamente métricas nutricionales basadas en los datos ingresados, incluyendo Tasa Metabólica Basal (TMB), Gasto Energético Total Diario (TDEE), y rangos de Proteínas, Carbohidratos y Grasas, ajustadas al nivel de actividad física y objetivos del usuario, con fórmulas detalladas a continuación:

**1. TMB (Tasa Metabólica Basal):** Para calcular el TMB se usará la fórmula de Mifflin-St Jeor, la cual es:

**Hombres**: TMB = (**10** × peso en kg) + (**6.25** × altura en cm) - (**5** × edad en años) + **5**

**Mujeres**: TMB = (**10** × peso en kg) + (**6.25** × altura en cm) - (**5** × edad en años) - **161**

### 2. TDEE (Gasto Energético Total Diario): Para calcular el TDEE se usará la fórmula de Mifflin-St Jeor ajustado por actividad, la cual es: {#tdee-gasto-energético-total-diario-para-calcular-el-tdee-se-usará-la-fórmula-de-mifflin-st-jeor-ajustado-por-actividad-la-cual-es}

**Paso 1: TDEE =** TMB × Factor de Actividad

Donde Factor de actividad es equivalente a un valor numérico que se elige en base a la actividad física de la persona a la cual se aplica el cálculo:

- Sedentario (poco o nada de ejercicio): TMB × **1.2**

- Ligeramente activo (ejercicio ligero 1-3 días/semana): TMB × **1.375**

- Moderadamente activo (ejercicio moderado 3-5 días/semana): TMB × **1.55**

- Muy activo (ejercicio intenso 6-7 días/semana): TMB × **1.725**

- Extremadamente activo (atletas, trabajos físicos intensos): TMB × **1.9**

### Paso 2: Ajustar Calorías Según Objetivo

- **Mantenimiento:** Usa el TDEE tal cual.

- **Pérdida de peso:** Resta 15-20% del TDEE. (A elección del nutriólogo)

- **Ganancia muscular:** Suma 10-20% al TDEE.(A elección del nutriólogo)

### 3. PROTEÍNAS:  {#proteínas}

**Paso 1: Proteínas =** peso corporal(kg) × Factor de Actividad física

**Sedentario:** 0.8 - 1.0 g/kg

**Ligeramente activo:** 1.0 - 1.2 g/kg

**Moderadamente activo:** 1.2 - 1.5 g/kg

**Muy activo:** 1.5 - 2.0 g/kg

**Extremadamente activo:** 2.0 - 2.5 g/kg

**4. GRASAS**: Se usará el rango recomendado de grasas al día por la OMS

**Grasas mínimas** = TDEE x 0.20 / 9

**Grasas máximas** = TDEE x 0.35 / 9

**5. CARBOHIDRATOS:** Se usará el rango recomendado de carbohidratos al día por las OMS

**Carbohidratos mínimos** = (TDEE x 0.45) / 4

**Carbohidratos máximos** = (TDEE x 0.65) / 4

> **15**. **Módulo de Control de Acceso\_\_\_\_ aqui me quede**

### Propósito

Permitir a los suscriptores del sistema AxF GymNet acceder a cualquier sucursal de la red mediante autenticación con tecnología NFC o huella digital, utilizando un microcontrolador ESP32 programado en C para verificar sus registros en la base de datos y desactivar una cerradura magnética durante 10 segundos para permitir el ingreso, garantizando un acceso seguro, rápido y automatizado en todas las sucursales.

### Funciones Principales

- **Autenticar Acceso con NFC**: Permite a los suscriptores autenticarse en la entrada de la sucursal escaneando su tarjeta NFC en un lector PN532 conectado al ESP32. El ESP32 lee el identificador único de la tarjeta y lo envía al servidor Node.js, que verifica en la base de datos MySQL si la tarjeta está asociada a un suscriptor con una suscripción activa. Si la verificación es exitosa, el ESP32 desactiva la cerradura magnética durante 10 segundos para permitir el ingreso.

- **Autenticar Acceso con Huella Digital**: Habilita la autenticación mediante huella digital utilizando un sensor R307 conectado al ESP32. El suscriptor coloca su dedo en el lector, y el ESP32 captura la huella, enviándola al servidor Node.js para compararla con los registros almacenados en la base de datos MySQL. Si la huella coincide con un suscriptor con suscripción activa, el ESP32 desactiva la cerradura magnética durante 10 segundos para permitir el ingreso.

- **Controlar Cerradura Magnética**: Gestiona la desactivación de la cerradura magnética (12V o 24V) conectada al ESP32. Tras una autenticación exitosa (NFC o huella digital), el ESP32 activa un relé conectado a la cerradura, manteniéndola desactivada durante 10 segundos para permitir el acceso del suscriptor, tras lo cual la cerradura se reactiva automáticamente.

- **Registrar Historial de Accesos**: Almacena un registro de cada acceso en la base de datos MySQL, incluyendo el identificador del suscriptor, el método de autenticación (NFC o huella), la sucursal, la fecha y la hora. Este historial es accesible para el usuario Sucursal y el personal autorizado (staff) a través del Módulo de Gestión de Sucursal para monitoreo y auditoría.

- **Notificar Fallos de Acceso**: Mediante una LCD 16x2 se mostrará si tiene acceso o no el suscriptor que intente acceder, este le indicara si no reconoce la huella, si la reconoce y no tiene suscripción activa, o si la reconoce y tiene suscripción activa, este último caso es el unico que permitirá el acceso.

### Detalles Técnicos de Implementación

- **Hardware Utilizado**:

  - **ESP32**: Microcontrolador programado en C utilizando Arduino IDE o ESP-IDF, encargado de gestionar la comunicación con los sensores NFC y de huella digital, y controlar la cerradura magnética.

  - **Módulo NFC PN532**: Lector/escritor NFC compatible con tarjetas MIFARE, conectado al ESP32 mediante interfaz SPI o I2C para leer identificadores únicos.

  - **Módulo R307**: Sensor de huella digital que captura y compara huellas mediante comunicación UART con el ESP32, soportando hasta 1000 plantillas de huellas.

  - **Cerradura Magnética**: Dispositivo electromagnético (12V/24V) controlado por un relé conectado a un pin GPIO del ESP32, que desactiva la cerradura durante 10 segundos tras autenticación exitosa.

  - **Relé**: Interruptor que controla la cerradura magnética, activado por el ESP32 para gestionar el flujo de corriente.

- **Flujo de Datos**:

  - El suscriptor presenta su tarjeta NFC o huella digital en el lector PN532 o R307, respectivamente.

  - El ESP32 captura el identificador NFC o la plantilla de huella y los envía al servidor Node.js mediante una solicitud HTTPS a través de Wi-Fi integrado.

  - El servidor Node.js verifica los datos contra la base de datos MySQL, comprobando si el identificador NFC o la huella corresponde a un suscriptor con suscripción activa.

  - Si la autenticación es exitosa, el servidor envía una respuesta al ESP32, que activa el relé para desactivar la cerradura magnética durante 10 segundos.

  - El acceso se registra en la base de datos MySQL con los detalles correspondientes (suscriptor, método, sucursal, fecha, hora).

  - En caso de fallo, el servidor notifica al ESP32, que activa un indicador (LED o buzzer) y registra el intento en el historial.

- **Seguridad**:

  - Los identificadores NFC y plantillas de huellas digitales se almacenan en la base de datos MySQL con encriptación (por ejemplo, AES-256 para identificadores NFC).

  - La comunicación entre el ESP32 y el servidor Node.js utiliza HTTPS para proteger los datos transmitidos.

  - El módulo R307 almacena plantillas de huellas digitales de forma segura, sin guardar imágenes crudas de las huellas.

- **Escalabilidad**:

  - El sistema permite configurar múltiples dispositivos ESP32 por sucursal, cada uno asociado a una puerta específica.

  - La base de datos MySQL soporta miles de registros de acceso, escalable para redes de gimnasios con múltiples sucursales.

##  16. Módulo de Análisis de Incidencias {#módulo-de-análisis-de-incidencias}

### Propósito

Permitir al sistema AxF GymNet analizar automáticamente los datos de los reportes de incidencias generados por los suscriptores a través del Módulo de Aplicación Móvil para Suscriptores, generando informes que pueden ser configurados en cuanto a frecuencia, osea que puedes seleccionar la frecuencia con la que deseas recibirlos, esto directamente lo hace el usuario sucursal en el modulo de gestion de sucursal, que se envían al usuario Sucursal a través de la interfaz web. Este módulo identifica patrones, mide tiempos de resolución, evalúa la frecuencia de incidencias, todas las notificaciones se mandan mediante una campanita presente en el sitio web que agrega un número marcado en rojo junto a ella indicando el número de notificaciones, en caso de no haber solo se ve la campanita sin ningún indicador de números.

### Funciones Principales

- **Generar Informe de Incidencias**:  
  > Crea un informe automatizado que se envía al usuario Sucursal a través del sistema web, accesible desde el Módulo de Gestión de Sucursal. El informe incluye:

  - **Incidencias Más Comunes**: Identifica las categorías de incidencias más frecuentes (por ejemplo, Máquina Dañada, Baño Tapado, Problema de Limpieza, Otro), mostrando la información en una gráfica de barras para separar las categorías del total de incidencias.

  - **Tiempos de Resolución**: Calcula el tiempo promedio, mínimo y máximo de resolución para cada categoría de incidencia, desde el momento del reporte hasta que se marca como resuelto. Incluye un análisis de reportes con \"strikes\" para destacar aquellos con tiempos de resolución prolongados, es decir, incluye el número de strikes en cada categoría, mostrando la información en una tabla.

  - **Distribución por Sucursal:** Muestra la cantidad de incidencias por sucursal, basándose en la selección de sucursal realizada por el suscriptor al generar el reporte mediante la lista desplegable en la aplicación móvil. Esto permite comparar el rendimiento entre diferentes ubicaciones y se presenta en una gráfica de barras.

  - **Tasa de Resolución**: Calcula el porcentaje de incidencias resueltas frente a las pendientes, proporcionando una gráfica de pastel que muestra el % resuelto y el % sin resolver, al dar click en la porción de "resuelto" se desplegará una lista con los reportes resueltos, y al dar click la porción de "sin resolver", aparecerán todos los reportes sin resolver igualmente en una lista.

  - **Prioridad de Reportes**: Destaca los reportes con \"tercer strike\" o reenviados directamente al usuario Sucursal, indicando la urgencia de atención, esto en una lista con todos los reportes que entren en la categoría.

  - **Reportes de Personal:** El módulo generará una gráfica de pastel para cada nutriólogo y entrenador, mostrando la proporción y numero de reportes recibidos de suscriptores que han recibido servicios personalizados (nutrición o entrenamiento, según corresponda) frente a aquellos que no han recibido dichos servicios. Estas gráficas, accesibles desde la interfaz del Módulo de Gestión de Sucursal, se generarán automáticamente con los informes periódicos y permitirán al usuario Sucursal analizar la relación entre los reportes y los servicios prestados por cada empleado

- **Visualizar Informe en la Interfaz Web**:  
  > Presenta el informe en una sección dedicada dentro del Módulo de Gestión de Sucursal.

- **Exportar Informe en PDF**:  
  > Ofrece un botón para generar un documento PDF con el contenido del informe, incluyendo gráficos y métricas, para referencia o auditoría. El PDF se estructura con un encabezado que indica el período analizado, la sucursal y la fecha de generación.

- **Enviar Notificaciones Automáticas**:  
  > Configura alertas automáticas que notifique al usuario Sucursal cuando se genera un nuevo informe.

- **Historial de Informes**:  
  > Mantiene un registro de todos los informes generados, accesible desde el Módulo de Gestión de Sucursal, con detalles como fecha de generación, período analizado y métricas clave, permitiendo al usuario Sucursal consultar informes anteriores para auditorías o comparaciones.

**Descripción de Usuarios del Sistema AxF GymNet**

El sistema AxF GymNet ha sido diseñado para optimizar las operaciones de una red de gimnasios, atendiendo a las necesidades específicas de 3 perfiles de usuario clave: el Usuario Maestro, el Usuario Sucursal y los diversos puestos dentro del Usuario Personal los cuales son: Staff, Entrenadores, Nutriólogos Deportivos y el rol combinado de Nutriólogo/Entrenador.

**1. Usuario Maestro (Administrador de la Red): El Arquitecto de la Operación**

- **Perfil:** Representa la autoridad central de la red de gimnasios, ya sea el propietario individual o un equipo de gestión corporativa. Su enfoque principal es la supervisión estratégica y la administración de la infraestructura general del negocio.

- **Interacción con el Sistema:**

  - **Gestión Integral de Sucursales:** Interactúa principalmente con el Módulo de Gestión de Sucursales. Aquí, puede crear nuevas sucursales, ingresando información crucial como nombre, dirección, código postal y las credenciales iniciales (usuario y contraseña) para el futuro administrador de esa sucursal. También tiene la potestad de modificar la información de las sucursales existentes, eliminarlas del sistema en caso de cierre o reestructuración, y buscar rápidamente los detalles de cualquier sucursal activa para obtener una visión clara de la red.

  - **Mantenimiento y Configuración:** Si bien no se explicita en la descripción de módulos, es probable que el Usuario Maestro tenga acceso a configuraciones generales del sistema, como la gestión de los tipos de suscripciones base que pueden ofrecerse en las sucursales o la administración de la integración con la API de pagos de Mercado Pago a nivel de red.

- **Beneficios Específicos:**

  - **Eficiencia Administrativa:** Simplifica la gestión de múltiples ubicaciones a través de una única plataforma.

  - **Escalabilidad:** Facilita la expansión de la red al permitir la creación y gestión eficiente de nuevas sucursales.

**2. Usuario Sucursal (Administrador de Sucursal): El Director de la Unidad Operativa**

- **Perfil:** Actúa como el enlace entre la dirección general y el equipo operativo.

- **Interacción con el Sistema:**

  - **Gestión del Personal Local:** Interactúa intensamente con las funcionalidades de administración de personal dentro del Módulo de Gestión de Sucursal. Aquí, puede dar de alta a nuevos empleados: Staff, Entrenadores, Nutriólogos y Nutriólogos/Entrenadores, asignándoles roles específicos a través de un menú desplegable en el campo \"puesto\". Al crear un nuevo usuario personal, ingresa información detallada como nombres, apellidos, edad, usuario, contraseña, sexo y la sucursal a la que pertenecen. También puede modificar la información de los empleados existentes, eliminarlos del sistema en caso de baja y buscar rápidamente sus perfiles.

  - **Administración de Servicios Locales:** Es el principal encargado de la gestión de las suscripciones y promociones ofrecidas en su sucursal. A través de interfaces específicas dentro del Módulo de Gestión de Sucursal, puede crear nuevos tipos de suscripciones (definiendo duración, nombre y precio), modificarlos, eliminarlos y consultarlos. De manera similar, gestiona las promociones (definiendo duración, nombre, descripción y precio).

  - **Comunicación Interna Local:** Utiliza la función de envío de avisos para comunicarse directamente con el personal de su sucursal, seleccionando destinatarios específicos para asegurar que la información relevante llegue a los empleados correctos.

- **Datos que Maneja:** información personal, roles y credenciales de acceso, los tipos de suscripciones y promociones ofrecidas localmente.

- **Beneficios Específicos:**

  - **Control Localizado:** Tiene autonomía para gestionar el personal y los servicios específicos de su sucursal.

  - **Comunicación Directa:** Facilita la comunicación interna con el personal, asegurando una operación más fluida.

**3. Usuario Personal: La Fuerza Operativa del Gimnasio**

Todos los usuarios dentro de esta categoría inician sesión en el sistema a través del Módulo de Inicio de Sistema, donde se autentican con sus credenciales y el sistema identifica su sucursal y rango para filtrar su acceso a los módulos pertinentes. Desde la pantalla principal, tienen acceso directo a los módulos de \"Usuarios\" y \"Gestionar plan de suscripción\".

- **3.1. Staff: El Primer Contacto y Soporte Administrativo  
  > **

  - **Perfil:** Personal de apoyo operativo cuya función principal es la interacción inicial con los suscriptores y la gestión administrativa básica relacionada con ellos.

  - **Interacción con el Sistema:**

    - **Gestión de Suscriptores:** Utilizan el Módulo de Usuarios para registrar nuevos suscriptores en el sistema, ingresando información personal detallada y utilizando los dispositivos de hardware conectados (lector de huellas y sensor NFC) para registrar la huella digital y la tarjeta NFC del usuario, respectivamente. También pueden modificar la información de los suscriptores existentes, buscarlos por nombre o apellidos y eliminarlos del sistema.

    - **Gestión de Membresías:** A través del Módulo de Gestión de Suscripciones, pueden verificar el estado de la suscripción de un usuario, procesar cancelaciones y facilitar el proceso de pago de nuevas suscripciones (aunque la transacción real se realiza a través de la integración con Mercado Pago).

  - **Datos que Maneja:** Información personal de los suscriptores, estado de sus suscripciones y detalles de los pagos (aunque no procesan directamente la información bancaria).

  - **Beneficios Específicos:**

    - **Registro Eficiente de Usuarios:** Agiliza el proceso de alta de nuevos miembros con la captura digital de datos y la vinculación de métodos de acceso.

    - **Gestión Simplificada de Membresías:** Facilita la consulta y gestión del estado de las suscripciones.

    - **Mejora en la Atención al Cliente:** Permite un acceso rápido a la información del suscriptor para brindar un mejor servicio.

- **3.2. Entrenadores: Los Arquitectos del Fitness Personalizado  
  > **

  - **Perfil:** Profesionales del fitness encargados de diseñar y guiar los programas de entrenamiento de los suscriptores.

  - **Interacción con el Sistema:** Además de las funcionalidades del Staff, los entrenadores utilizan el Módulo de Entrenamiento para:

    - **Cargar Ejercicios:** Ingresan nuevos ejercicios a la base de datos, incluyendo el nombre y una imagen de referencia. Esta función enriquece la biblioteca de ejercicios disponibles para la creación de rutinas.

    - **Crear Rutinas:** Diseñan planes de entrenamiento personalizados seleccionando ejercicios de la base de datos, especificando series, repeticiones, descansos y, opcionalmente, el peso recomendado. Pueden agregar notas técnicas sobre la ejecución de los ejercicios y comentarios generales para el suscriptor. Finalmente, pueden enviar la rutina por correo electrónico al usuario o generar un PDF para su impresión o envío.

  - **Datos que Maneja:** Información personal y de suscripción de los suscriptores (al igual que el Staff), así como datos relacionados con los ejercicios (nombres e imágenes) y las rutinas de entrenamiento (series, repeticiones, descansos, pesos y notas).

  - **Beneficios Específicos:**

    - **Planificación Eficiente de Entrenamientos:** Facilita la creación de rutinas personalizadas utilizando una base de datos de ejercicios.

    - **Comunicación Directa con el Suscriptor:** Permite enviar las rutinas de forma digital o impresa.

    - **Profesionalización del Servicio:** Ofrece planes de entrenamiento estructurados y personalizados.

- **3.3. Nutriólogos Deportivos: Los Guías de la Alimentación Saludable  
  > **

  - **Perfil:** Profesionales de la nutrición especializados en el ámbito deportivo, responsables de diseñar planes de alimentación para los suscriptores.

  - **Interacción con el Sistema:** Además de las funcionalidades del Staff, los nutriólogos utilizan el Módulo de Nutrición para:

    - **Registros de Usuarios:** Consultan la lista de suscriptores activos y acceden a una interfaz para registrar y actualizar datos relevantes sobre su estado físico (peso, altura, edad, porcentaje de grasa y músculo, actividad física, objetivo). El sistema calcula automáticamente métricas como TMB y TDEE, así como los rangos recomendados de macronutrientes. También pueden agregar notas sobre el estado del usuario y consultar un historial de registros.

    - **Cargar Recetas:** Ingresan nuevas recetas a la base de datos, incluyendo una descripción detallada, una imagen y los valores nutricionales (proteínas, calorías y grasas). Esto enriquece la base de datos de recetas disponibles para la creación de dietas.

    - **Crear Dietas:** Diseñan planes de alimentación diarios seleccionando recetas precargadas o agregando alimentos manualmente para cada comida. Pueden agregar notas específicas para cada comida y generar un PDF con el plan de dieta para los días completados.

  - **Datos que Maneja:** Información personal y de suscripción de los suscriptores (al igual que el Staff), datos biométricos y de actividad física de los usuarios, información nutricional de las recetas y los planes de dieta diseñados.

  - **Beneficios Específicos:**

    - **Planificación Nutricional Personalizada:** Facilita la creación de dietas basadas en las necesidades y objetivos individuales de los suscriptores.

    - **Acceso a Información Nutricional:** Proporciona una base de datos de recetas con valores nutricionales.

    - **Seguimiento del Progreso Nutricional:** Permite registrar y consultar la evolución de los usuarios.

- **3.4. Nutriólogo/Entrenador: El Profesional Integral del Bienestar  
  > **

  - **Perfil:** Individuos con la doble certificación y capacidad para ofrecer tanto planes de entrenamiento como planes de nutrición personalizados.

  - **Interacción con el Sistema:** Tienen acceso y utilizan todas las funcionalidades disponibles para el Staff, los Entrenadores y los Nutriólogos Deportivos. Esto les permite gestionar la información de los suscriptores y sus membresías, crear rutinas de entrenamiento personalizadas, registrar datos biométricos, cargar recetas y diseñar planes de alimentación integrales.

  - **Datos que Maneja:** La totalidad de los datos manejados por el Staff, los Entrenadores y los Nutriólogos Deportivos.

  - **Beneficios Específicos:**

    - **Servicio Integral:** Pueden ofrecer un servicio completo y coordinado de entrenamiento y nutrición a los suscriptores.

    - **Eficiencia y Comodidad:** Simplifican la comunicación y la planificación al ser un único punto de contacto para el suscriptor en ambas áreas.

### Descripción de las funciones y características que tiende

El sistema contará con una amplia gama de funciones y características diseñadas para resolver los problemas identificados y cumplir con los objetivos plantados. Estas se agrupan en categorías clave:

#### Funciones Administrativas

- **Dirección de Sucursales:** Creación, edición y eliminación de productos por el Usuario Maestro.

- **Dirección de Personal:** Registro y administración de usuarios "personal" por el Usuario Sucursal.

- **Dirección de Membresías:** Registro, renovación y seguimiento de suscripciones con integración de APIs de pago (Mercado Pago).

#### Funciones Operativas para el Personal

- **Rutinas Personalizadas:** Creación de rutinas de ejercicio en PDF con registro histórico de métricas (1RM, PR, etc.).

- **Dietas Personalizadas:** Generación de planes alimenticios en PDF basados en datos del suscriptor (peso, objetivos, preferencias).

- **Seguimiento de Progreso:** Monitoreo de avances físicos y nutricionales con registros históricos accesibles.

#### Funciones de Acceso y Seguridad

- **Control de Acceso:** Autenticación mediante huella digital y NFC con esp32

- **Seguridad de Datos:** Encriptación de contraseñas y protección de datos sensibles (personales y de salud) en la base de datos.

#### Funciones de Informes

- **Generación de PDFs:** Exportación de rutinas y dietas personalizadas en formato PDF para uso interno y entrega a suscriptores.

- **Reportes Administrativos:** Estadísticas de miembros, accesos y rendimiento por sucursal.

#### Características Técnicas

- **Disponibilidad:** 24/7, accesible desde cualquier computadora del gimnasio con conexión a internet.

- **Compatibilidad:** Funciona en navegadores modernos (Chrome, Firefox, Safari, Edge) sin necesidad de software adicional.

- **Diseño:** Optimizado para pantallas grandes (PC), no responsive para móviles.

- **Integración:** Autosuficiente, con herramientas internas para gestión, bases de datos y pagos, sin depender de sistemas externos más allá de la API de pago y el hardware Arduino.

- **Usabilidad:** Interfaz intuitiva con capacitación incluida para el personal, asegurando una rápida adopción.

#### Beneficios Esperados

- **Eficacia:** Centralización de procesos que reducen la carga administrativa.

- **Personalización:** Planes de ejercicio y nutrición adaptados a cada suscriptor.

- **Seguridad:** Control de acceso y protección de datos sensibles.

- **Experiencia del Suscriptor:** Servicio más ágil y profesional gracias a la automatización y seguimiento detallado.

Para el desarrollo del sistema AxF GymNet y la aplicación móvil para suscriptores, se empleará un conjunto de tecnologías modernas y robustas que garantizarán la funcionalidad, escalabilidad y eficiencia del proyecto:

  - **Tailwind CSS**: Se implementará este framework de CSS \"utility-first\" para un **desarrollo rápido y personalizable de la interfaz**, permitiendo crear diseños modernos con clases predefinidas.

Kotlin es el lenguaje preferido por Google para el desarrollo Android y permite crear aplicaciones de alto rendimiento y con una excelente experiencia de usuario.

- **XML (Extensible Markup Language)**: Se utilizará para **definir los layouts y la estructura de la interfaz de usuario (UI)** de la aplicación móvil Android. XML permite diseñar las pantallas, botones, campos de texto y otros elementos visuales con los que interactuarán los suscriptores.

### Hardware e Integraciones Específicas

- **ESP32 y Lenguaje C**: Se utilizará el microcontrolador ESP32, programado en C, para gestionar el **Módulo de Control de Acceso**. Esto incluye la lectura de datos de sensores NFC y de huella digital, y la comunicación con el sistema principal para la autenticación y activación de la cerradura magnética.

- **API de Mercado Pago**: Se integrará para **gestionar los pagos de suscripciones en línea de forma segura y eficiente**, delegando la complejidad del procesamiento de transacciones financieras.

### Alojamiento

- **Hostinger**: Se utilizará como el **servicio de alojamiento web (hosting)** para la aplicación web y la base de datos MySQL, buscando un plan que ofrezca los recursos necesarios para la disponibilidad y rendimiento del sistema.
